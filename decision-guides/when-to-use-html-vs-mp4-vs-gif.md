# When To Use HTML vs MP4 vs GIF

## Use HTML Video Teaser When

- You want a presentation-quality animation as a link.
- You want crisp text and SVG, not compressed video.
- You need timeline control and scrubability.
- You want to host it on GitHub Pages, a website, or an internal page.
- You need a self-contained demo that can be edited in code.
- You are showing fake UI, product concepts, data flows, or agent workflows.

Do not use HTML teaser when the final deliverable must be uploaded as a video file to social or ad platforms.

## Use MP4 When

- The platform expects video upload.
- You need sound, captions, and final playback consistency.
- The piece will be used externally, at events, or in paid/social media.
- You need to hand it to non-technical stakeholders as a finished asset.

Use Remotion, After Effects, Premiere, CapCut, or another real video workflow.

## Use GIF When

- It is a very short silent loop.
- The goal is quick documentation or chat sharing.
- Quality can be lightweight.
- The loop is under 6 seconds.

Avoid GIF for text-heavy content. A short MP4 loop is often better.

## Hybrid Workflow

You can combine formats:

1. Storyboard in markdown.
2. Prototype motion as HTML or slides.
3. Screen-record/export to MP4.
4. Cut down to GIF or social clips.

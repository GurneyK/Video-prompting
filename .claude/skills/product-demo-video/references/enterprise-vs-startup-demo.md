# Enterprise vs. Startup Demo

Same feature, same underlying storyboard shape (`demo-narrative-patterns.md`), materially different execution. Confirm the lens with `creative-director/references/audience-lenses.md` before choosing.

## Enterprise (Unilever-style buyer)

- **Open with context that names the stakes, not the user.** "Every regional team was reconciling this manually across 40 markets" lands harder than "Meet Priya."
- **Include at least one proof beat aimed at a non-user stakeholder** — security, compliance, audit trail, or ROI framing. The end user liking the feature isn't enough; the demo needs to answer "why would IT/legal/finance approve this."
- **Palette and type:** restrained, professional — avoid playful color unless it matches an existing approved brand system. Longer holds (1.5-2s minimum) after key screens — the room reviewing this wants to read every number, not feel a vibe.
- **Data:** conservative, realistic-looking enterprise scale (multi-market, multi-thousand row counts, currency formats per market) — see `data-and-dashboard-fakery.md`.
- **Sound:** must have a clean captioned/silent version; background music should be understated, not energetic.
- **Length:** can afford to run longer (60-90s) if every beat earns its place — this audience is in a considered-purchase mindset, not scrolling past.
- **CTA:** usually "request a demo" / "talk to sales," rarely a self-serve signup.

## Startup (self-serve / PLG buyer)

- **Open with the hook in the first 2 seconds** — the surprising result, not the setup. Assume silent autoplay on a feed.
- **Skip the stakeholder-proof beat** unless selling into an enterprise buyer (see hybrid note below) — a startup audience wants "does this work for me," not organizational risk mitigation.
- **Palette and type:** can be bold, saturated, opinionated — this is often where the brand identity gets defined *through* the demo rather than applied to it.
- **Data:** can be smaller-scale and more personal/specific (a single realistic use case) rather than enterprise-scale numbers.
- **Sound:** music-forward is fine and often expected; UI click SFX read as satisfying rather than distracting at this pace.
- **Length:** short — 20-45s. If it's not clear by second 5 what this does, most of the audience is already gone.
- **CTA:** usually "try it free" / "sign up" / "start now" — low-friction, self-serve.

## Hybrid: startup selling to enterprise buyers (common B2B SaaS case)

- Keep startup production speed and pacing for the first half (hook fast, show the workflow fast).
- Add one enterprise-proof beat before the CTA — a security/compliance/ROI line — even if the rest of the demo feels startup-paced. This single beat is often the difference between an enterprise buyer trusting a fast-moving vendor or not.
- CTA can be dual: "Start free" for individual users evaluating, with a smaller secondary "Talk to sales" for the buyer who needs procurement involved.

```text
Before generating final output:
1. Ask missing questions.
2. Recommend format: MP4, animated presentation, HTML teaser, or Figma prototype.
3. Create a storyboard with scene timing.
4. Identify which UI should be real screenshots vs faux UI.
5. Write the production prompt or implementation plan.
6. Include QA for text readability, product accuracy, and confidentiality.
```

## Product Demo Story Pattern

1. User problem
2. Current friction
3. New workflow
4. Key interaction
5. Result/proof
6. CTA or next step

## QA

- UI labels match the product
- No fake claims or invented metrics
- Confidential data is removed
- Cursor motion has intent
- Viewer understands the outcome without narration

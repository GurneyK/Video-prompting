# Narrative Structures

Pick the shape before writing scenes. A storyboard built on the wrong shape will fight itself no matter how good the individual scenes are.

## 1. Problem → Tension → Proof → Transformation → CTA

The default for product marketing, launch teasers, and video ads.

| Beat | Job | Typical length share |
|---|---|---|
| Problem | Name the pain in the audience's own language, not the product's | 15% |
| Tension | Make the cost of the status quo felt, not just stated | 15% |
| Proof | Show the product actually doing the thing — this is the demo core | 40% |
| Transformation | The after-state — what's different now | 20% |
| CTA | One clear next action | 10% |

Best for: video ads, product launch teasers, homepage hero videos.

## 2. Before / After

Compressed two-beat version of #1 for very short formats (loops, 15-30s social).

`Before state (2-4s)` → `hard cut or morph` → `After state (2-4s)` → optional CTA card.

The cut/morph itself should be the "aha" — the transition IS the message. Best for GIF loops, 6-15s social cuts, feature-reveal stingers.

## 3. Hero's Journey (compressed)

For brand films, culture/recruiting content, founder stories, customer-story testimonials.

`Ordinary world` → `call to action / problem arrives` → `struggle / attempt` → `turning point (often where the product/idea appears)` → `new equilibrium`. Compress the middle two beats hard — most business use of this arc fails by over-indulging the "struggle" beat. Keep it to a single vivid moment, not a montage.

Best for: customer testimonials, culture/recruiting films, founder-story startup content.

## 4. Feature → Benefit → Proof

For demo videos and feature-reveal content where the audience already knows the product category and just needs convincing this feature matters.

`Show the feature (what it looks like)` → `state the benefit (why it matters to them)` → `prove it (real workflow, real number, real before/after)`. Repeat per feature, 3 features max per video — more than that and nothing lands. Order features by "most likely to be the reason they buy," not by build order in the product.

Best for: `product-demo-video` skill output, feature-reveal teasers, sales enablement clips.

## 5. Data-Story Arc

For spec-heavy reveals (hardware, performance benchmarks, infrastructure, anything with a number as the hero).

`Context for the number (why should I care)` → `reveal the number with maximum visual weight` → `comparison (vs. before, vs. competitor, vs. expectation)` → `what this unlocks for the viewer`. The number should appear exactly once at full weight — don't recycle the same stat as a running ticker, it dilutes the reveal.

Best for: SpaceX/Tesla-style spec reveals, performance benchmark launches, infra/dev-tool announcements. Pair with `templates/company-styles/spacex.md` or `templates/company-styles/tesla.md` for the visual register, and the `dataviz` skill for the chart/number construction itself.

## 6. Config/Community Keynote Arc

For developer-conference-style reveals and community/ecosystem content — a string of smaller "and one more thing" reveals rather than one big arc.

`Warm cold-open (community/energy, not product)` → `reveal 1 (small, sets tone)` → `reveal 2 (bigger)` → `reveal 3 (the headline one)` → `community/ecosystem close (not a hard CTA — a "come build with us")`. Energy should climb across reveals, not stay flat.

Best for: startup product launches with an ecosystem/community angle, internal all-hands "what shipped this quarter" videos. Pair with `templates/company-styles/figma.md` or `templates/company-styles/linear.md`.

## Choosing between them

| If the request is... | Use |
|---|---|
| A launch teaser or ad with a clear CTA | Problem→Tension→Proof→Transformation→CTA |
| A single feature, very short, social-first | Before/After |
| A testimonial, culture piece, or founder story | Hero's Journey (compressed) |
| A multi-feature demo video | Feature→Benefit→Proof |
| A hardware/performance/infra reveal | Data-Story Arc |
| A developer/community/quarterly-update reveal | Config/Community Keynote Arc |

If two seem to fit, pick the one that requires fewer scenes — the shape that gets to proof fastest usually wins.

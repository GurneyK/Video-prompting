# Synthesizing a Style for a Startup With No Design Team

The goal here is speed with just enough opinion to be usable — not a complete design system. A startup that waits for a "proper" brand system before shipping its first launch video has already lost the speed advantage that's supposed to be its edge (see `creative-director/references/audience-lenses.md` startup lens). This process should take minutes, not days.

## The fast process

1. **Pick one accent color, not a palette.** One color that will appear in every piece of launch content (a CTA, a highlight, an accent line). Near-black or near-white for the base, one accent color, done. Resist the urge to pick "a palette" — a single consistent accent is more distinctive at this stage than a sophisticated multi-color system nobody will apply consistently anyway.
2. **Pick one type pairing.** A display face for headlines/big claims (can be a strong system font — Inter, general sans-serif, or whatever the product's own UI already uses, for free consistency) and one body face, which can be the same family at a lighter weight. Don't introduce a third typeface.
3. **Pick one motion signature and write it down in one sentence.** E.g. "Fast, confident, `outCubic` entrances, hard cuts between scenes, no hold longer than 2 seconds." This single sentence is enough to keep a multi-scene video internally consistent.
4. **Decide the sound stance in one line.** Music-forward and energetic, or clean/silent-first with UI click SFX only — pick one, don't mix registers within one video.
5. **Write 3 Do's and 3 Don'ts, specific to this launch,** not generic brand principles. E.g. "Don't use stock gradient orbs. Do use the product's real UI in every scene that can show it."

## Where to steal from (properly)

It's completely legitimate at this stage to say "we want something like [reference company]'s energy" — that's exactly what `reference-company-styles.md` and `templates/company-styles/` are for. Pick one reference company whose *pacing and restraint level* (not literal visuals) matches the intended brand feel, use its style-DNA notes as a starting point, then swap in this startup's own color/type/copy. The output should read as "inspired by," never as "a reskin of."

## What NOT to do at this stage

- Don't hire the process out to a lengthy brand workshop before the first video ships — this system is deliberately meant to be revisited and upgraded later once there's real content to react to.
- Don't skip writing anything down because "we'll just know it when we see it" — even this minimal system prevents the second video looking like a different company made it.
- Don't borrow a reference company's exact palette or type — see `reference-company-styles.md` for the hard line between inspiration and reproduction.

## Revisiting later

Once the startup has shipped 2-3 pieces of content against this minimal system and it's working, that's the right moment to expand it into the full `style-pack-template.md` schema with a proper design pass — not before. Treat this fast version as v0.1, explicitly, so nobody mistakes it for a final system.
